1.白化uni-app

1.html标签和uni-app内置组件的映射表

html uni-app
div view
span、font text
a navigator
img image
input radio radio
input checkbox checkbox
input time时间 date日期 piker
select piker
form、button、label、textarea、canvas、video form、button、label、textarea、canvas、video
iframe web-view
ul、li view，做列表一般使用uList组件
audio 改成api方式，背景音频api文档
原html规范中input不仅是输入框，还有radio、checkbox、时间、日期、文件选择功能，在uni-app和小程序规范中，input仅仅是输入框

2.新增了一批手机端常用的新组件

标签 说明
scroll-view 可区域滚动视图容器
swiper 可滑动区域视图容器（轮播图）
icon 图标
rich-text 富文本（不可执行js，但可渲染各种文字格式和图片）
progress 进度条
slider 滑块指示器
switch 开关选择器
camera 页面内嵌的区域相机组件
live-player 直播
map 地图
cover-view 覆盖在原生组件上的文本视图。
audio 改成api方式，背景音频api文档
uni-app的非h5端的video、map、canvas、textarea是原生组件，层级高于其他组件。如需覆盖原生组件，则需要使用cover-view组件。

3.js的变化

js的变化，分为运行环境变化、数据绑定模式变化、api变化3部分。

1.运行环境从浏览器变成v8引擎
标准js语法和api都支持，比如if、for、settimeout、indexOf等。
但浏览器专用的window、document、navigator、location对象，包括cookie等存储，只有在浏览器中才有，app和小程序都不支持。
可能有些人以为js等于浏览器里的js。其实js是ECMAScript组织管理的，浏览器中的js是w3c组织基于js规范补充了window、document、navigator、location等专用对象。

2.以前的dom操作，改成vue的MVVM模式
如果你想改变某个dom元素的显示内容，比如一个view的显示文字：
以前是给view设id，然后js里通过选择器获取dom元素，进一步通过js进行赋值操作，修改dom元素的属性或值。
vue的绑定模式，给这个dom元素绑定一个js变量，在script中修改js变量的值，dom会自动变化，页面会自动更新渲染

3.js api的变化

alert,confirm 改成 uni.showmodel
ajax 改成 uni.request
cookie、session 没有了，local.storage 改成 uni.storage
uni-app的js api还有很多，但基本就是小程序的api，把wx.xxx改为uni.xxx即可。

4.css的变化

选择器有2个变化：*选择器不支持；元素选择器里没有body，改为了page。微信小程序即是如此。
单位方面，如果想使用根据屏幕宽度自适应的单位，推荐使用rpx，全端支持。
uni-app推荐使用flex布局，这种布局更灵活高效且支持更多平台。
注意css里背景图和字体文件，尽量不要大于40k，因为会影响性能。

5.工程结构和页面管理

每个可显示的页面，都必须在 pages.json 中注册。

2.发布与运行

1.运行
运行App到手机或模拟器：使用电压足够的usb端口连接手机，设置中开启USB调试，手机上允许电脑设备调试手机，进入hello-uniapp项目，点击工具栏的运行 -> 运行App到手机或模拟器，即可在该设备里面体验uni-app。
在微信开发者工具里运行：进入hello-uniapp项目，点击工具栏的运行 -> 运行到小程序模拟器 -> 微信开发者工具，即可在微信开发者工具里面体验uni-app。
注：
1.微信开发者工具需要开启服务端口 在微信工具的设置->安全中。
2.微信小程序工具需要配置允许权限，不然HBuilder无法调用微信小程序开发工具的命令行
3.如果自动启动小程序开发工具失败，请手动启动小程序开发工具并将 HBuilderX 控制台提示的项目路径，打开项目。
https://uniapp.dcloud.net.cn/quickstart-hx.html

2.发布为微信小程序：
在HBuilderX中顶部菜单依次点击 “发行” => “小程序-微信”，输入小程 序名称和appid点击发行即可
hbuildx 发行 微信小程序 发行
微信开发者工具 上传
微信公众平台 管理 版本管理 开发版本 选为体验版本 点击二维码分享

3.跨端注意

1.H5正常但小程序异常的可能性

1.v-html在h5和app-vue均支持，但小程序不支持
2.小程序要求连接的网址都要配白名单
3.css异常：

不支持的选择器
非H5端不支持*选择器；
body的元素选择器请改为page，同样，div和ul和li等改为view、span和font改为text、a改为navigator、img改为image…

组件和页面样式相互影响
非H5端默认并未启用 scoped，如需要隔离组件样式可以在 style 标签增加 scoped 属性，H5端为了隔离页面间的样式默认启用了 scoped

2.使用 Vue.js 的注意

uni-app 基于Vue 2.0实现
data 属性必须声明为返回一个初始数据对象的函数；否则页面关闭时，数据不会自动销毁，再次打开该页面时，会显示上次数据

3.H5 开发注意

1.CSS 內使用 vh 单位的时候注意 100vh 包含导航栏，使用时需要减去导航栏和 tabBar 高度，部分浏览器还包含浏览器操作栏高度，使用时请注意。
2.正常支持 rpx，px 是真实物理像素。暂不支持通过设 manifest.json 的 “transformPx” : true，把 px 当动态单位使用。
3.使用罗盘、地理位置、加速计等相关接口需要使用 https 协议，本地预览（localhost）可以使用 http 协议。
4.组件内（页面除外）不支持 onLoad、onShow 等页面生命周期。
