css文本

1.书写模式

writing-mode: CSS 中的书写模式是指文本的排列方向是横向还是纵向的。
取值:
horizontal-tb: 块流向从上至下。对应的文本方向是横向的。
vertical-rl: 块流向从右向左。对应的文本方向是纵向的。
vertical-lr: 块流向从左向右。对应的文本方向是纵向的。

2.块级布局和内联布局

当我们切换书写模式时，我们也在改变块和内联文本的方向。
因此，块维度指的总是块在页面书写模式下的显示方向。而内联维度指的总是文本方向。

水平书写模式下的两种维度。

纵向书写模式下的两种维度。

3.逻辑属性和逻辑值

横向书写模式下，映射到width的属性被称作内联尺寸（inline-size）——内联维度的尺寸。而映射height的属性被称为块级尺寸（block-size），这是块级维度的尺寸。

映射对应(物理 –> 逻辑属性)

1.逻辑属性
width –> inline-size
height –> block-size
margin-top –> inline-size
padding-left –> padding-inline-start
border-bottom –> border-block-end

2.逻辑值
top –> block-start
bottom –> block-end
left –> inline-start
right –> inline-end

如果用writing-mode把盒子.box的书写模式改为vertical-rl，将会看到尽管盒子的物理方向变了，盒子的物理属性仍然没变，然而逻辑属性会随着书写模式一起改变。

值和单位

1.长度

1.绝对长度单位
1cm = 37.8px(像素)
2.相对长度单位

单位 相对于
em 相对于当前对象内文本的font-size，如果当前文本的字体尺寸没有设置，则相对于浏览器的默认字体尺寸。即1em=16px。
rem 根元素的字体大小。
vw 视口宽度的 1%。
vh 视口高度的 1%。

3.em 和 rem

em：相对于当前对象内文本的font-size，如果当前文本的字体尺寸没有设置，则相对于浏览器的默认字体尺寸。

rem：相对HTML根元素。

em和rem的区别
①rem是相对于根元素进行计算，而em是相对于当前元素或父元素的字体大小，一直追溯到根元素。
②rem不仅可以设置字体的大小，还支持元素宽、高等属性。
③em是相对于当前元素或父元素进行换算，层级越深，换算越复杂。而rem是相对于根元素计算，避免层级关系。

2.数字

CSS中，每个属性可以接受哪些值都有不同的规定，有的属性只能接受预定义的值，有的属性接受数字、长度、百分数、URL或颜色，有的属性还可以接受多种类型的值。

CSS中有两类数字：整数和浮点数
CSS属性接受不带单位的数字，最常见的就是 line-height、z-index、opacity，如line-height: 1.5。

3.颜色

rgba()和rgb()、十六进制RGB、颜色关键词、hls() 和 hlsa()

4.图片

<image> 数据类型用于图像为有效值的任何地方。

5.位置

<position> 数据类型表示一组 2D 坐标，用于定位一个元素，如背景图像（通过 background-position）
它可以使用关键字（如 top、left、bottom、right 以及 center）将元素与 2D 框的特定边界对齐，以及表示框的顶部和左侧边缘偏移量的长度。

6.字符串和标识符

关键字被用作值的地方（例如 <color> 关键字，如 red、black、rebeccapurple 和 goldenrod）。这些关键字被更准确地描述为标识符，一个 CSS 可以理解的特殊值。因此它们没有使用引号括起来——它们不被当作字符串。

在某些地方可以使用 CSS 中的字符串，例如在指定生成的内容时。
