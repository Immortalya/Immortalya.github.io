1.文本处理

1.1. 标题

h1-h6 依次减小

1.有序

1.3. 强调

粗体：<blockquote> <b>、斜体：<em> <i>
下标：<sub>、上标：<sup>
放大：<big>、缩小：<small>
高亮：<mark>(html5)
删除文字: <del>、下划线：<ins>

1.缩略语：<abbr>
用来包裹一个缩略语或缩写，并且提供缩写的解释。
<abbr title="超文本标记语言（Hyper text Markup Language）">HTML</abbr>

2.标记联系方式:<address>
样式为斜体，和i标签一样

3.计算机代码
<code>：用于标记计算机通用代码。
<pre>：用于保留空白字符（通常用于代码块）——如果文本中使用了缩进或多余的空白，浏览器将忽略它，你将不会在呈现的页面上看到它。
<var>：用于标记具体变量名。
<kbd>：用于标记输入电脑的键盘（或其他类型）输入。
<samp>：用于标记计算机程序的输出。

1.4. 引用

1.快引用
如果一个块级内容（一个段落、多个段落、一个列表等）从其他地方被引用，你应该把它用

元素包裹起来表示，并且在 cite 属性里用 URL 来指向引用的资源。

浏览器的默认样式会将其呈现为缩进的段落，作为引用的指示

2.行内引用

浏览器默认将其作为普通文本放入引号内表示引用

3.引文
<cite> 元素包含所引用资源的标题（如书名）
引文默认的字体样式为斜体。

1.5. 其他文本元素

字符实体：空格(&nbsp;) <(<) >(>) &(&)
html编码：0(48) a(97) A(65)
水平线：hr(换行符可自定义样式)、 换行符：br
改变文字方向：从左到右(ltr)、从右到左(rtl)

2.超链接

<a href="https://www.runoob.com" target="" title="了解站点。">文本或图像</a>
1.href
超链接所指向的 URL。链接不限于基于 HTTP 的 URL——它们可以使用浏览器支持的任何 URL 协议
例如：链接到同一页面的不同位置

2.target
用于指定链接页面的打开方式，取值有_self, _blank, _top, _parent
_self ：在同一框架或窗口中打开所链接的文档。此参数为默认值，通常不用指定
_blank ：在新浏览器窗口中打开链接文件
_top ：在当前的整个浏览器窗口中打开所链接的文档，因而会删除所有框架。
_parent ：将链接的文件载入含有该链接框架的父框架集或父窗口中。如果含有该链接的框架不是嵌套的，则在浏览器全屏窗口中载入链接的文件，就象 _self 参数一。

3.title
当鼠标指针悬停在链接上时，标题将作为提示信息出现

4.download
导致浏览器将链接的 URL 视为下载资源。可以使用或不使用 filename 值
注：download 只在同源 URL 或 blob:、data: 协议起作用。
<a href="/images/logo.png" download="/images/logo.png" filename="自定义文件名">下载</a>

5.onclick 事件
锚点元素经常被滥用为假按钮，将其 href 设置为 # 或 javascript:void(0) 以防止页面刷新，然后监听其 click 事件。

3.文档与网站架构

为了实现语义化标记，HTML 提供了明确这些区段的专用标签
header 是简介形式的内容。如果它是 body 的子元素，那么就是网站的全局页眉。如果它是 article 或section 的子元素，那么它是这些部分特有的页眉（此 header 非彼 标题）。
nav 包含页面主导航功能。其中不应包含二级链接等内容。
main 存放每个页面独有的内容。每个页面上只能用一次，且直接位于 body 中。最好不要把它嵌套进其他元素。
article 包围的内容即一篇文章，与页面其他部分无关（比如一篇博文）。
section 与 article 类似，但 section 更适用于组织页面使其按功能（比如迷你地图、一组文章标题和摘要）分块。一般的最佳用法是：以 标题 作为开头；也可以把一篇 article 分成若干部分并分别置于不同的 section 中，也可以把一个区段 section 分成若干部分并分别置于不同的 article 中，取决于上下文。
aside 包含一些间接信息（术语条目、作者简介、相关链接，等等）。
footer 包含了页面的页脚部分。
hgroup 标签用于对网页或区段的标题元素（h1-h6）进行组合

3.元素之间的转换问题：
display:inline; —> 把元素转换成行内元素 —> 不独占一行，不能设置宽高
display:inline-block; 把元素转换成行内块元素 —>不独占一行，可以设置宽高
display:block; —> 把元素转换成块元素 —> 独占一行，且可以设置宽高 display:none; 不显示

2. p元素不能包含任何块级元素(包括自身)
3. a元素可以包含任何其他元素(除了自身)
4. document是文档(整个DOM树)的根节点
5. li元素的祖先元素可能是li，但父元素不可能是li

4.多媒体与嵌入

4.1. 图片

img属性
src: 图像路径
alt：对图片的文字描述，用于在图片无法显示或不能被看到的情况
width/height：宽高
title：鼠标悬停时显示的内容
border：图像边框的宽度
align：图像对齐到left、right、top、middle、bottom
vspace：设置图像顶部和底部的空白
hspace：设置图像左侧和右侧的空白

<img src="/images/logo.png" alt="文本" title="文本" width="258" height="39" border="数字" align="" vspace="" hspace="" />

1.添加一段说明文字来搭配图片

这个 <figcaption> 元素说明文字描述了 <figure> 元素的内容
figure 可以是几张图片、一段代码、音视频、方程、表格或别的。

4.2. 视频和音频

1.video 视频

video属性
src: 指向你想要嵌入网页当中的视频资源
controls: 控件界面
width和height: 宽高
autoplay: 使音频和视频内容立即播放
loop: 让音频或者视频文件循环播放
muted: 媒体播放时，默认关闭声音
poster: 指向了一个图像的 URL，这个图像会在视频播放前显示
preload: 这个属性被用来缓冲较大的文件。none：不缓冲；auto：页面加载后缓存媒体文件；metadata：仅缓冲文件的元数据

2.音频 audio

audio 除了width/height、poster属性，支持所有

4.3. iframe

1.iframe属性
src:指向要嵌入文档的 URL 路径
width 和 height:指定 iframe 的宽度和高度。
allowfullscreen: 可以通过全屏 API 设置为全屏模式
frameborder: 如果设置为 1，则会在此框架和其他框架之间绘制边框，这是默认行为。0 删除边框。
sandbox:需要在已经支持其他 iframe 功能（>= IE10）但稍微更现代的浏览器上才能工作

2.<frameset> 标签：frameset 定义一个框架集，包含多个子框架frame，每个框架都有独立的文档
使用frameset标签时，注意要去掉外层的<body></body>标签。

frameset 属性：
　　rows ：表示子框架按行的样式布局。以2个子框架为例：rows=”30%,“ ，表示第一个框架占整个页面30%的高度，第二个占剩下的；
　　cols ：表示子框架按列的样式布局。以2个子框架为例：cols=”30%,“ ，表示第一个框架占整个页面30%的长度，第二个占剩下的；
noresize=”noresize” ：表示不调整子框架的范围。

frame 属性：
　　src ：指向一个资源(如页面、图片等)的URI；
　　name ：指定框架的名称，以便进行框架间的操作。
