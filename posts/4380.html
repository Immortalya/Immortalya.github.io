指令 (Directives) 是带有 v- 前缀的特殊 attribute。

1.v-text：填充纯文本

2.v-html：更新元素的 innerHTML

3.v-pre：用来显示原始 Mustache 标签。

Mustache是Javascript模板引擎mustache.js

<span v-pre>{{ this will not be compiled }}</span>
输出：{{ this will not be compiled }}

4.v-once：一次性地插值，当数据改变时，插值处的内容不会更新

5.v-model：双向数据绑定，常用于在表单控件或者组件上

6.v-bind：主要用于属性绑定

动态地绑定一个或多个 attribute，或一个组件 prop 到表达式。
在绑定 class 或 style attribute 时，支持其它类型的值，如数组或对象。
在绑定 prop 时，prop 必须在子组件中声明。可以用修饰符指定不同的绑定类型。
简写：:

动态绑定class

1.对象语法
①给v-bind:class 一个对象，以动态地切换class。注意：v-bind:class指令可以与普通的class特性共存

②也可以直接绑定数据里的一个对象

2.数组语法
①可以把一个数组传给v-bind:class，以应用一个class列表

②如果想根据条件切换列表中的class，可以用三目运算

③对于多个class，可以这么写：
<div :class="[classA, { classB: isB, classC: isC }]">
<p :class="{class1:true,class2:flag==true,class3:flag==false} ">联系我们</p>

④用在组件上
当在一个自定义组件上使用 class property 时，这些 class 将被添加到该组件的根元素上面。这个元素上已经存在的 class 不会被覆盖。

</my-component>组件 将被渲染为：<p class="foo bar baz boo">Hi</p>

绑定内联样式style

v-bind:style 的对象语法十分直观–非常像CSS，其实它是一个Javascript对象

也可以直接绑定到一个样式对象或者数组上，这里就不多赘述了。

7.v-show：根据表达式的真假值，控制元素样式是否显示（已经渲染到页面）。

8.v-if、v-else-if、v-else

v-if：控制元素是否渲染到页面
在切换时元素及它的数据绑定 / 组件被销毁并重建。如果元素是 <template>，将提出它的内容作为条件块。

切换多个元素, 可以把一个 <template> 元素当做不可见的包裹元素，并在上面使用 v-if。最终的渲染结果将不包含 <template> 元素。

用 key 管理可复用的元素

Vue 会尽可能高效地渲染元素，通常会复用已有元素而不是从头开始渲染。

Vue 提供了一种方式来表达“这两个元素是完全独立的，不要复用它们”。只需添加一个具有唯一值的 key attribute 即可

现在，每次切换时，输入框都将被重新渲染。此时 <label>元素仍然会被高效地复用。

9.v-for：基于源数据多次渲染元素或模板块。

此指令之值，必须使用特定语法 alias in expression，为当前遍历的元素提供别名

要强制其重新排序元素，你需要用特殊 attribute key 来提供一个排序提示

预期：Array | Object | number | string | Iterable (2.6 新增)

注：当和 v-if 一起使用时，v-for 的优先级比 v-if 更高。

1.数组更新检测

Vue 将被侦听的数组的变更方法进行了包裹，所以它们也将会触发视图更新。
增删改排序：push()、pop()、shift()、unshift()、splice()、sort()、reverse()

注：由于 JavaScript 的限制，Vue 不能检测数组和对象的变化。

2.显示过滤/排序后的结果

1.可以创建一个计算属性，来返回过滤或排序后的数组。

2.在计算属性不适用的情况下 (例如，在嵌套 v-for 循环中) 你可以使用一个方法

3.在组件上使用 v-for

在自定义组件上，你可以像在任何普通元素上一样使用 v-for。
<my-component v-for="item in items" :key="item.id"></my-component>

然而，任何数据都不会被自动传递到组件里，因为组件有自己独立的作用域。为了把迭代数据传递到组件里，要使用 prop

10.v-on: 绑定事件监听器。

事件类型由参数指定。表达式可以是一个方法的名字或一个内联语句，如果没有修饰符也可以省略

①事件修饰符
.stop - 调用 event.stopPropagation()。
.prevent - 调用 event.preventDefault()。
.capture - 添加事件侦听器时使用 capture 模式。
.self - 只当事件是从侦听器绑定的元素本身触发时才触发回调。
.{keyCode | keyAlias} - 只当事件是从特定键触发时才触发回调。
.native - 监听组件根元素的原生事件。
.once - 只触发一次回调。
.left - (2.2.0) 只当点击鼠标左键时触发。
.right - (2.2.0) 只当点击鼠标右键时触发。
.middle - (2.2.0) 只当点击鼠标中键时触发。
.passive - (2.3.0) 以 { passive: true } 模式添加侦听器

11.v-slot：提供具名插槽或需要接收 prop 的插槽。

限用于：<template>、组件 (对于一个单独的带 prop 的默认插槽)

12.v-cloak：处理插值表达式闪动
